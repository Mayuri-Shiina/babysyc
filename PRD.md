# 宝宝成长档案 Agent App PRD v1

## 1. 文档目标
本文档用于明确宝宝成长档案 Agent App 的 v1 产品定位、用户角色、核心功能、Agent 能力、权限边界、技术约束与验收范围，作为后续设计、研发和迭代的统一依据。

## 2. 产品概述

### 2.1 产品定位
这是一个面向单家庭私有场景的宝宝成长档案 App，以情感记录为主，辅以轻量智能能力，帮助家庭长期保存宝宝成长过程中的照片、视频、里程碑、成长记录和阶段总结。

产品核心不是做泛化育儿社区，也不是做医疗诊断工具，而是做一个可沉淀、可回看、可整理、可分享的私人家庭档案系统。

### 2.2 产品一句话描述
一个帮助家庭记录宝宝成长、自动整理回忆并结合真实档案提供提醒与建议的私有育儿助手。

### 2.3 v1 核心原则
- 情感记录优先，智能能力服务于记录与整理。
- 默认私有，仅受邀成员或受控分享对象可见。
- 单家庭优先设计，但数据模型保留未来多家庭、多宝宝扩展能力。
- 医疗边界清晰，仅提供健康建议、风险提示和就医建议，不提供诊断结论。

## 3. 目标与非目标

### 3.1 产品目标
- 帮助家庭低门槛地持续记录宝宝成长过程。
- 将零散照片、视频、文字和体征记录整理成统一时间轴。
- 通过 Agent 自动生成成长总结，减少人工整理成本。
- 通过轻提醒和引导，降低“想记录但不知道记什么”的门槛。
- 为家庭成员提供安全、稳定、可控的共同查看与编辑能力。

### 3.2 v1 非目标
- 不做开放社区、动态广场、公开社交分享。
- 不做医生端或医疗机构协同系统。
- 不做正式医疗器械级评估或疾病诊断。
- 不做原生 iOS / Android App，v1 采用 Web + PWA。
- 不接入微信登录、短信登录、视频理解。

## 4. 目标用户与角色

### 4.1 核心使用者
- `SuperOwner`：你本人，既是项目开发者，也是宝宝舅舅，拥有系统内最高权限。
- `Editor`：宝宝父母、外公外婆、爷爷奶奶、叔叔等近亲，可参与日常记录与编辑。
- `Viewer`：被邀请的人或拿到分享链接的人，只能查看授权内容。
- `Anonymous`：未受邀、未持有分享凭证的陌生人，无任何权限。

### 4.2 使用场景
- 家人上传宝宝照片、视频和日常记录。
- 家人记录身高、体重、睡眠、喂养、体温等成长数据。
- 家人记录第一次微笑、翻身、出门等里程碑事件。
- 家人查看时间轴，回顾某一阶段的成长变化。
- Agent 自动总结宝宝近期成长情况，并提醒缺失记录或关键节点。

## 5. 权限模型

### 5.1 角色权限定义
| 角色 | 说明 | 主要权限 |
| --- | --- | --- |
| `SuperOwner` | 系统最高权限，默认唯一 | 管理家庭、成员、权限、宝宝档案、全部记录、分享设置、系统配置 |
| `Editor` | 近亲协作者 | 新增、编辑、删除成长记录、里程碑、照片视频、说明文字、提醒备注 |
| `Viewer` | 受邀只读访客 | 只读查看被授权内容，不可新增、编辑、删除 |
| `Anonymous` | 未授权陌生人 | 无权限 |

### 5.2 权限原则
- 默认私有，陌生人不可访问任何宝宝数据。
- `Viewer` 仅通过邀请或分享链接获得访问权限。
- 分享能力需受控，至少支持有效期与内容可见范围控制。
- 后续如需增加“禁止下载原图/视频”或“访客水印”等能力，作为增强项规划，不纳入 v1 必做范围。

## 6. 产品价值主张

### 6.1 对家庭的价值
- 留住成长回忆，而不是只堆积素材。
- 让全家共同参与记录，但权限边界清晰。
- 让记录形成长期可回看的成长档案。

### 6.2 对使用体验的价值
- 不是“想到才记”，而是被轻柔引导去记录。
- 不是“素材太多难整理”，而是 Agent 帮忙总结、归类、生成回忆卡片。
- 不是“泛泛而谈的育儿问答”，而是基于当前宝宝真实档案的上下文回答。

## 7. 信息架构

### 7.1 一级模块
v1 采用 5 个一级模块：
- `首页`
- `成长记录`
- `疫苗`
- `相册/视频`
- `Agent 助手`

### 7.2 首页定位
首页是“家庭档案入口”，整体体验应优先传达温暖、回忆感和正在成长中的状态，而不是冷冰冰的数据看板。

首页建议包含：
- 宝宝今日卡片：日龄、月龄、头像、当天一句话摘要
- 最近照片/视频
- 本周成长总结
- 最新成长记录
- 待提醒事项
- 今日建议记录内容

### 7.3 时间轴定位
时间轴是产品最核心的内容容器，应支持以下内容混排：
- 照片
- 视频
- 文字记录
- 成长体征记录
- 里程碑卡片
- Agent 生成的周报/月报总结卡片

## 8. 核心功能需求

### 8.1 宝宝档案
#### 功能描述
创建并维护宝宝基础档案，作为所有记录和 Agent 上下文的基础。

#### 基础字段
- 宝宝昵称
- 出生日期
- 出生时间
- 性别
- 出生地点
- 出生身高 / 体重
- 备注信息

#### v1 要求
- 支持创建一个当前宝宝档案。
- 数据模型预留多宝宝扩展能力。
- 首页和 Agent 默认围绕当前宝宝展开。

### 8.2 成长记录
#### 功能描述
记录宝宝日常成长数据，并自动生成基础趋势分析。

#### 记录类型
- 身高
- 体重
- 头围
- 体温
- 睡眠
- 喂养
- 备注

#### v1 要求
- 支持新增、编辑、删除记录。
- 自动计算记录时的日龄和月龄。
- 保留 BMI 展示，但分析主依据采用更适合婴幼儿的年龄/性别生长曲线百分位。
- 每次录入后生成简短分析：偏低、正常、偏高、趋势变化、是否建议复测或咨询医生。

### 8.3 里程碑记录
#### 功能描述
记录宝宝成长中的关键第一次和重要节点，强化情感价值。

#### 示例里程碑
- 第一次微笑
- 第一次抬头
- 第一次翻身
- 第一次出门
- 第一次辅食
- 第一颗牙

#### v1 要求
- 支持手动创建里程碑。
- 支持上传照片或视频并与里程碑关联。
- 支持 Agent 为里程碑生成简短标题或说明文案。

### 8.4 相册 / 视频
#### 功能描述
管理宝宝成长过程中的图片和视频素材，支撑时间轴和回忆整理。

#### v1 要求
- 支持家庭成员上传图片和视频。
- 支持按时间轴浏览。
- 支持按月份筛选。
- 支持手动打标签和事件关联。
- 数据库存储元数据，对象文件存储在 S3 兼容存储中。

### 8.5 疫苗与提醒
#### 功能描述
管理疫苗计划和关键提醒，但不喧宾夺主，作为辅助能力存在。

#### v1 要求
- 默认按中国国家免疫规划生成时间表。
- 支持记录已接种、延期、补种、禁忌与医生建议。
- 支持疫苗提醒、体检提醒、记录缺失提醒、阶段节点提醒。
- 通知方式采用站内提醒 + Web Push。

## 9. Agent 能力设计

### 9.1 能力定位
Agent 在本产品中不是独立主角，而是家庭档案的整理员、提示员和解读员。所有 Agent 能力都应围绕“记录、整理、提醒、总结”展开。

### 9.2 v1 核心能力

#### 1. 成长总结助手
核心目标：帮助家庭自动沉淀回忆。

输入来源：
- 最近一周或一月的成长记录
- 新增照片与视频
- 新增里程碑
- 疫苗与提醒状态

输出内容：
- 本周 / 本月成长摘要
- 本阶段新增里程碑
- 最近值得回看的照片或事件
- 下阶段值得关注的记录建议

输出形式：
- 首页总结卡片
- 可保存到时间轴的总结卡片

#### 2. 记录引导助手
核心目标：降低记录门槛，提升持续记录率。

典型触发：
- 多天未更新成长记录
- 某月龄阶段建议关注新发育表现
- 上传照片后未填写描述
- 某类记录长期缺失

输出内容：
- 今天建议记录什么
- 最近缺少哪些记录
- 当前月龄可关注哪些变化

#### 3. 宝宝上下文问答助手
核心目标：围绕当前宝宝真实档案回答问题。

回答应结合：
- 宝宝日龄 / 月龄
- 最近成长记录
- 最近里程碑
- 最近照片标签
- 疫苗与提醒状态

适合的问题示例：
- 最近一周睡眠变化怎么样
- 这阶段适合记录哪些表现
- 最近体重增长趋势如何
- 接下来有哪些关键提醒

### 9.3 输入模态
v1 支持：
- 文本
- 图片
- 语音

v1 不支持：
- 视频理解
- 医疗诊断

### 9.4 回答结构规范
Agent 回答统一采用三层结构：
- `结论`
- `原因`
- `建议行动`

### 9.5 医疗安全边界
- 遇到高风险症状关键词时，触发明显风险提示。
- 直接建议尽快线下就医或急诊。
- 不输出诊断结论，不替代儿科医生建议。

## 10. 关键用户流程

### 10.1 日常记录流程
1. 家人打开首页。
2. 查看今日卡片和记录建议。
3. 上传照片、填写说明或新增成长记录。
4. 系统自动关联时间、宝宝上下文和提醒状态。
5. Agent 在后续生成总结或补充记录引导。

### 10.2 周报生成流程
1. 系统按周聚合新记录、新照片和里程碑。
2. Agent 生成本周成长总结。
3. 家人可查看、保存、再次编辑或加入时间轴。

### 10.3 分享查看流程
1. `SuperOwner` 或具备授权能力的角色生成分享链接。
2. 设置访问期限和内容可见范围。
3. 受邀对象以 `Viewer` 身份查看授权内容。

## 11. 核心数据模型
- `Family`：家庭空间。
- `User`：用户账户。
- `FamilyMember`：用户在家庭内的角色与权限。
- `Baby`：宝宝基础档案。
- `GrowthRecord`：成长原始记录。
- `GrowthAssessment`：成长分析结果。
- `DevelopmentMilestone`：里程碑与发育建议。
- `VaccinePlan` / `VaccineRecord`：疫苗计划与接种记录。
- `MediaAsset`：图片/视频元数据。
- `Reminder`：提醒事项。
- `AgentSession` / `AgentMessage`：Agent 对话与风险标记。

所有核心数据应按 `family_id` 和 `baby_id` 建模，以保留未来多家庭、多宝宝扩展能力。

## 12. 技术方案约束

### 12.1 前端
- `Next.js App Router`
- `TypeScript`
- `Tailwind CSS`
- `PWA`

### 12.2 后端
- `FastAPI`
- 模块拆分：`auth`、`family`、`baby-profile`、`growth`、`vaccine`、`media`、`reminder`、`agent`

### 12.3 数据与存储
- `PostgreSQL`：主数据库
- `Redis`：会话缓存、任务队列、提醒调度
- `S3 兼容对象存储`：图片和视频文件
- `Celery + Redis`：异步任务
- `pgvector`：内置知识库与 FAQ 检索

### 12.4 AI 模型能力约束
- 通过可替换的 OpenAI 兼容网关接入。
- 至少支持多模态聊天和语音转写。
- 家庭私有上下文不得跨家庭共享。

### 13.1 v1 必做
- 宝宝档案
- 成长记录
- 里程碑记录

## 14. 验收标准

### 14.1 权限验收
- `SuperOwner` 可管理全部内容和成员权限。
- `Editor` 可正常新增、编辑、删除记录。
- `Viewer` 仅能查看授权内容。
- `Anonymous` 无法访问任何宝宝数据。

### 14.2 记录验收
- 成长记录可新增、编辑、删除。
- 录入身高体重后可正确计算年龄、BMI、百分位和状态。
- 里程碑可关联照片/视频并进入时间轴。

### 14.3 媒体验收
- 图片/视频上传成功。
- 时间轴可按时间顺序查看。
- 权限校验正确，未授权用户不可查看。

### 14.4 提醒验收
- 疫苗、体检、记录缺失、发育节点提醒可正常触发。
- 站内提醒和 Web Push 可用。

### 14.5 Agent 验收
- 支持文本、图片、语音三类输入。
- Agent 回答能带入宝宝年龄和近期记录。
- 可生成周报或月报总结卡片。
- 高风险症状提问时输出风险提示，不输出诊断性结论。

### 14.6 PWA 验收
- 手机浏览器可正常访问。
- 可安装到桌面。
- 基础离线壳与推送订阅可用。

## 15. 风险与待决策项
- 需要后续明确分享链接的最小安全策略，避免隐私暴露。
- 需要明确成长曲线百分位的数据源和计算标准。
- 需要明确周报/月报的文风边界，确保温暖但不过度煽情。
- 需要明确语音转写和图片理解的成本控制方案。

## 16. 当前结论
v1 应坚持“私人家庭档案”定位，以时间轴、里程碑、成长总结为产品主心骨，让 Agent 服务于记录、整理和轻量决策支持，而不是喧宾夺主地变成独立聊天产品。
