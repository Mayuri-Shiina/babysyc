# 宝宝成长档案 Agent App MVP v1

## 1. MVP 目标
MVP 的目标不是一次性做完整产品，而是尽快验证这个项目最核心的价值是否成立：

- 家庭是否愿意持续记录宝宝成长内容。
- 记录内容是否能被整理成有温度的成长档案。
- Agent 是否能真正提升记录效率和回顾体验。

MVP 必须先跑通一个最小闭环：

`添加宝宝 -> 邀请家人 -> 上传照片/记录成长 -> 自动生成总结 -> 家人查看和回顾`

## 2. MVP 产品定义

### 2.1 一句话版本
一个让家庭共同记录宝宝成长，并自动生成阶段总结的私有成长档案 App。

### 2.2 MVP 核心定位
- 主体是“成长档案”而不是“聊天机器人”。
- 重点是“情感记录 + 自动整理”，不是复杂育儿工具集合。
- Agent 在 MVP 中只做最有价值的辅助能力，不追求全能。

## 3. MVP 用户与权限

### 3.1 用户角色
- `SuperOwner`：你本人，最高权限。
- `Editor`：宝宝父母和指定亲属，可编辑内容。
- `Viewer`：受邀只读访客或持有效分享链接的访问者。
- `Anonymous`：无权限。

### 3.2 MVP 权限范围
- `SuperOwner`：创建家庭、创建宝宝、邀请成员、管理权限、管理所有记录。
- `Editor`：新增、编辑、删除照片、视频、成长记录、里程碑、文字说明。
- `Viewer`：只读浏览已授权内容。

## 4. MVP 必须解决的问题
- 家人如何方便地上传和记录内容。
- 记录如何统一沉淀到一条时间轴中。
- 家人如何快速看到“最近宝宝发生了什么”。
- 系统如何提醒“该记录什么”，而不是等用户自己想起来。

## 5. MVP 功能范围

### 5.1 必做功能

#### 1. 宝宝档案
- 创建一个宝宝档案。
- 支持昵称、生日、出生时间、性别、出生信息等基础字段。
- 系统自动计算日龄和月龄。

#### 2. 邀请与权限
- `SuperOwner` 可邀请家庭成员。
- 成员角色可设置为 `Editor` 或 `Viewer`。
- 未授权用户不可访问任何内容。

#### 3. 相册 / 视频上传
- 支持上传照片。
- 支持上传视频。
- 支持填写简短描述。
- 支持按时间展示。

#### 4. 成长记录
- 支持录入身高、体重、头围、体温、睡眠、喂养、备注。
- 支持编辑和删除。
- 自动生成基础分析结果。

#### 5. 里程碑记录
- 支持手动创建里程碑。
- 支持关联照片或视频。
- 支持填写标题和说明。

#### 6. 时间轴
- 将照片、视频、成长记录、里程碑混排展示。
- 支持按时间倒序浏览。
- 支持按月份筛选。

#### 7. 首页
- 展示宝宝今日卡片。
- 展示最近照片。
- 展示最新记录。
- 展示本周成长总结。
- 展示待提醒事项和建议记录内容。

#### 8. Agent 能力
MVP 只做以下 3 个能力：

- `成长总结助手`
- `记录引导助手`
- `宝宝上下文问答助手`

其中优先级最高的是 `成长总结助手`。

#### 9. 轻提醒
- 记录缺失提醒。
- 关键月龄提醒。
- 疫苗提醒。

### 5.2 可以弱化实现的功能
- 疫苗模块先以基础计划和提醒为主，不做复杂规则引擎。
- 语音输入可先只打通上传和转写，不做复杂语音体验优化。
- 图片理解先只用于辅助标签和描述生成，不做复杂识别。

### 5.3 不纳入 MVP 的功能
- 原生 App
- 视频理解
- 社区分享
- 医疗诊断
- 多宝宝切换 UI
- 访客水印
- 原图防下载
- 复杂运营后台

## 6. MVP 核心页面

### 6.1 首页
核心任务：
- 让用户一打开就知道最近有什么新变化。
- 看到最新回忆和最近建议操作。

页面模块：
- 宝宝今日卡片
- 最近照片
- 本周成长总结
- 最新成长记录
- 待提醒事项
- 今日建议记录

### 6.2 时间轴页
核心任务：
- 统一承载宝宝成长过程中的所有内容。

页面模块：
- 按时间排序的混合卡片流
- 月份筛选
- 快速添加入口

### 6.3 成长记录页
核心任务：
- 录入和查看身高、体重、睡眠、喂养等记录。

页面模块：
- 快速录入表单
- 最近记录列表
- 简单趋势图

### 6.4 相册 / 视频页
核心任务：
- 浏览和补充媒体素材。

页面模块：
- 图片 / 视频网格
- 时间筛选
- 标签和描述编辑

### 6.5 Agent 页
核心任务：
- 集中承载问答、总结和记录建议。

页面模块：
- 问答输入框
- 最近总结卡片
- 记录建议列表

## 7. MVP Agent 设计

### 7.1 成长总结助手
目标：
- 自动整理过去 7 天或 30 天的成长内容。

最小输入：
- 新增照片和视频
- 新增成长记录
- 新增里程碑

最小输出：
- 一段成长摘要
- 2 到 4 条关键变化
- 下阶段记录建议

### 7.2 记录引导助手
目标：
- 告诉家人“今天值得记什么”。

最小触发：
- 3 天无记录
- 上传照片未写描述
- 当前月龄存在推荐记录点

最小输出：
- 1 到 3 条建议记录项

### 7.3 宝宝上下文问答助手
目标：
- 基于当前宝宝档案回答具体问题。

最小能力：
- 回答最近记录变化
- 回答当前月龄相关建议
- 回答接下来提醒事项

## 8. MVP 数据范围

### 8.1 必要数据实体
- `User`
- `Family`
- `FamilyMember`
- `Baby`
- `GrowthRecord`
- `DevelopmentMilestone`
- `MediaAsset`
- `Reminder`
- `AgentSession`
- `AgentMessage`

### 8.2 可延后数据实体
- `GrowthAssessment` 的复杂扩展字段
- 复杂疫苗规则明细表
- 更完整的知识库运营结构

## 9. MVP 验收标准

### 9.1 使用闭环验收
- 可创建宝宝档案。
- 可邀请至少 1 名 `Editor` 和 1 名 `Viewer`。
- `Editor` 可上传照片并新增成长记录。
- 时间轴可看到新增内容。
- 首页可展示最近内容和总结。
- Agent 可生成本周成长总结。

### 9.2 权限验收
- `SuperOwner` 可查看并管理全部内容。
- `Editor` 可编辑内容。
- `Viewer` 只可查看。
- 未授权访问被拒绝。

### 9.3 Agent 验收
- 可以基于真实记录生成总结。
- 可以给出记录建议。
- 可以回答与当前宝宝档案相关的问题。

## 10. MVP 建议开发顺序

### 阶段 1：基础骨架
- 项目初始化
- 登录 / 邀请 / 权限模型
- 宝宝档案
- 基础页面框架

### 阶段 2：记录闭环
- 照片视频上传
- 成长记录
- 里程碑记录
- 时间轴

### 阶段 3：智能闭环
- 成长总结助手
- 记录引导助手
- 宝宝上下文问答助手
- 首页总结与提醒整合

### 阶段 4：补全体验
- 疫苗提醒
- PWA
- Web Push
- 基础分享能力

## 11. 当前建议
MVP 不要试图一开始做成“全功能育儿平台”。第一版只要把“家庭记录 + 时间轴回顾 + Agent 自动总结”这条链路做顺，这个产品就已经成立了。
